 # Smart Building ML Classification Pipeline - Deskripsi Project

## Latar Belakang Project

Project Smart Building ML Classification Pipeline adalah sebuah sistem machine learning yang dirancang untuk melakukan deteksi anomali dan klasifikasi kesalahan (fault classification) pada dataset Smart Building. Dataset ini berisi data operasional dari bangunan pintar dengan dua kelas utama: data normal dan data yang mengalami gangguan/kesalahan. Tujuan utama project ini adalah membangun model machine learning yang mampu mengidentifikasi pola kesalahan dalam sistem bangunan dengan akurasi minimal 90%, sehingga dapat membantu dalam pemeliharaan preventif dan peningkatan efisiensi operasional bangunan.

## Dataset dan Struktur Data

Project menggunakan dataset Smart Building yang terdiri dari empat file CSV terpisah: normal_training.csv untuk data latih normal, faulty_training.csv untuk data latih yang mengalami kesalahan, normal_testing.csv untuk data uji normal, dan faulty_testing.csv untuk data uji yang mengalami kesalahan. Data ini telah dipisahkan ke dalam folder `/data/` dan siap digunakan oleh pipeline. Setiap file berisi fitur-fitur sensor dari bangunan pintar dengan label biner (0 untuk normal, 1 untuk faulty) yang memudahkan proses klasifikasi. Struktur data yang terorganisir ini memastikan tidak ada data leakage antara set pelatihan dan pengujian, sehingga evaluasi model dapat dilakukan dengan cara yang valid dan dapat dipercaya.

## Pipeline dan Workflow

Pipeline machine learning ini dirancang dengan alur kerja yang sistematis dan terstruktur dalam sebuah Jupyter Notebook dengan 40 cell yang mencakup semua tahap pengembangan model. Pertama, data dimuat dari keempat file CSV dan digabungkan dengan label yang sesuai. Kemudian dilakukan analisis eksploratori untuk memahami karakteristik dataset, termasuk distribusi kelas, statistik fitur, dan identifikasi outlier. Tahap preprocessing melibatkan penskalaan fitur menggunakan StandardScaler yang dipasang hanya pada data latih untuk mencegah data leakage. Setelah itu, fitur-fitur yang sangat berkorelasi dihapus dan feature selection dilakukan berdasarkan importance dari algoritma Random Forest. Untuk mengatasi ketidakseimbangan kelas, SMOTE (Synthetic Minority Over-sampling Technique) diterapkan hanya pada data latih untuk menghasilkan sampel sintetis dari kelas minoritas.

## Model dan Algoritma

Project mengimplementasikan dan mengevaluasi sembilan algoritma machine learning yang berbeda untuk menemukan model terbaik. Algoritma-algoritma tersebut mencakup Logistic Regression untuk pemodelan linear, K-Nearest Neighbors untuk klasifikasi berbasis jarak, Decision Tree sebagai model dasar ensemble, Random Forest sebagai ensemble method yang kuat, Gradient Boosting untuk peningkatan iteratif, XGBoost sebagai implementasi boosting yang efisien, LightGBM untuk training yang cepat, Support Vector Machine untuk klasifikasi non-linear, dan Naive Bayes sebagai probabilistic classifier. Setiap model dilatih pada data yang telah diseimbangkan menggunakan SMOTE untuk memastikan pembelajaran yang optimal dari kedua kelas. Variasi algoritma ini memungkinkan perbandingan komprehensif untuk mengidentifikasi pendekatan mana yang paling efektif untuk masalah deteksi anomali pada Smart Building.

## Evaluasi dan Metrik

Evaluasi model dilakukan menggunakan berbagai metrik yang komprehensif untuk memberikan gambaran lengkap tentang performa setiap algoritma. Metrik utama yang digunakan meliputi Accuracy untuk mengukur persentase prediksi yang benar, Precision untuk mengukur ketepatan prediksi positif, Recall untuk mengukur kemampuan menemukan kasus positif, dan F1-Score sebagai harmonic mean antara Precision dan Recall. Selain itu, ROC-AUC digunakan untuk mengevaluasi trade-off antara true positive rate dan false positive rate. Confusion matrix divisualisasikan untuk memberikan detail tentang true positives, true negatives, false positives, dan false negatives. Classification report memberikan metrik per-kelas yang detail. Untuk memastikan stabilitas model, dilakukan cross-validation dengan 5-fold untuk mengurangi variance dalam estimasi performa dan memvalidasi bahwa model generalisasi dengan baik pada data yang tidak terlihat.

## Output dan Hasil

Setelah menjalankan pipeline, sistem akan menghasilkan berbagai output yang komprehensif untuk dokumentasi dan analisis lebih lanjut. Output utama meliputi file CSV berisi metrik performa semua model (model_results.csv), file pickle untuk menyimpan model terbaik (best_model.pkl) dan scaler untuk preprocessing data baru (scaler.pkl), serta visualisasi dalam format PNG seperti confusion matrices untuk tiga model terbaik, comparison chart untuk semua metrik, correlation matrix untuk hubungan antar fitur, SMOTE balance visualization untuk menunjukkan efek resampling, dan cross-validation results untuk menunjukkan stabilitas model. Semua output disimpan di folder `/output/` untuk kemudahan akses dan dokumentasi. Dengan output yang terorganisir ini, pengguna dapat melakukan analisis mendalam tentang performa model dan membuat keputusan berdasarkan data untuk deployment atau improvement lebih lanjut.

## Fitur dan Best Practices

Project dibangun dengan menerapkan best practices dalam machine learning untuk memastikan hasil yang valid dan dapat dipercaya. Tidak ada data leakage yang terjadi karena scaler dipasang hanya pada data latih dan SMOTE hanya diterapkan pada training data, memastikan bahwa test set tetap sebagai evaluasi yang independent dan unbiased dari model. Reproducibility dijamin melalui pengaturan RANDOM_STATE = 42 di semua tempat yang relevan sehingga hasil eksekusi akan identik setiap kali menjalankan notebook. Pipeline dirancang dengan modular dan extensible, menggunakan utility scripts terpisah untuk berbagai fungsi seperti data exploration, model definition, evaluation, dan visualization. Dokumentasi yang komprehensif dalam bentuk file README.md memudahkan pengguna untuk memahami struktur project, menjalankan pipeline, dan menginterpretasi hasil. Error handling dan validation checks diintegrasikan di seluruh pipeline untuk mendeteksi dan memberikan informasi tentang potensi masalah.

## Teknologi dan Dependencies

Project dibangun menggunakan Python 3.7+ sebagai bahasa pemrograman utama dengan beberapa library penting yang terinstal melalui pip. Library pandas dan numpy digunakan untuk manipulasi dan operasi numerik data, scikit-learn menyediakan algoritma machine learning dan tools untuk preprocessing serta evaluation, xgboost dan lightgbm menghadirkan implementasi gradient boosting yang efisien dan cepat, imbalanced-learn (imblearn) menyediakan teknik SMOTE untuk mengatasi class imbalance, matplotlib dan seaborn digunakan untuk visualisasi hasil dan analisis data, dan jupyter digunakan untuk membuat dan menjalankan interactive notebook. Semua dependencies ini tersedia di PyPI (Python Package Index) dan dapat diinstal dengan mudah menggunakan perintah pip, membuat setup project menjadi straightforward dan dapat direplikasi di berbagai environment.

## Target dan Success Criteria

Target utama project adalah mencapai akurasi minimal 90% dalam mengklasifikasikan data Smart Building antara kondisi normal dan kondisi faulty/error. Pencapaian target ini akan menunjukkan bahwa model machine learning berhasil mempelajari pola yang membedakan kedua kelas dengan akurasi yang tinggi dan dapat diandalkan untuk aplikasi praktis. Jika target tercapai, model dianggap siap untuk dideploy ke production environment dengan confidence bahwa performanya akan konsisten pada data baru. Jika akurasi tetap berada di bawah target 90%, notebook akan memberikan rekomendasi spesifik untuk improvement seperti feature engineering lebih lanjut, pengumpulan data tambahan terutama dari kelas minoritas, atau eksplorasi hyperparameter tuning yang lebih agresif. Selain akurasi utama, success criteria sekunder mencakup precision dan recall yang seimbang untuk menghindari bias terhadap satu kelas, stabilitas model yang ditunjukkan oleh variance rendah dalam cross-validation scores, dan visualisasi yang jelas untuk memfasilitasi interpretasi hasil kepada stakeholders.

## Penggunaan dan Deployment

Untuk menggunakan project ini, pengguna hanya perlu membuka terminal, navigasi ke direktori project, menjalankan perintah `jupyter notebook`, membuka file `scripts/ML_Pipeline_Notebook.ipynb`, dan menjalankan semua cell untuk mengeksekusi seluruh pipeline (proses ini biasanya memakan waktu 5-15 menit tergantung spesifikasi hardware). Setelah eksekusi selesai, semua hasil akan tersimpan di folder `/output/` dan metrik performa akan ditampilkan di console. Model terbaik yang dihasilkan dapat disimpan dan digunakan untuk membuat prediksi pada data baru dengan menggunakan scaler yang telah disimpan untuk memastikan konsistensi preprocessing. Untuk deployment production, best_model.pkl dan scaler.pkl dapat dimuat menggunakan joblib dan diintegrasikan ke dalam aplikasi atau service yang lebih besar untuk real-time anomaly detection pada Smart Building system.

## Kesimpulan

Smart Building ML Classification Pipeline adalah solusi end-to-end yang komprehensif untuk deteksi anomali pada sistem bangunan pintar menggunakan machine learning. Project menggabungkan data yang terstruktur, preprocessing yang ketat, evaluasi yang teliti dari berbagai algoritma, dan output yang informatif untuk memandu pengambilan keputusan. Dengan menerapkan best practices seperti prevention of data leakage, reproducibility, dan cross-validation, project menghasilkan model yang valid dan dapat dipercaya untuk identifikasi kesalahan pada Smart Building. Pipeline ini dapat dijalankan dengan mudah oleh pengguna dengan berbagai tingkat keahlian, dan hasil yang dihasilkan memberikan insight yang jelas tentang performa model serta rekomendasi untuk improvement jika diperlukan, menjadikan project ini valuable tool untuk maintenance dan optimization operasional bangunan pintar.
